FlexChart

FlexChart is a collection of Wijmo chart components, with over 80 different chart types to choose from. There are three different groups of charts: standard charts, financial charts, and special charts. Special charts are more unique and serve different purposes that the other chart categories.

FlexChart, like all of Wijmo's components, uses the CollectionView to manage the data provided to the datagrid. This allows devs extensive control over how the data is organized and presented by FlexGrid. It can also allow controls to be collected to one another by having them use the same CollectionView.

This page will outline some of FlexChart's key charts and features.

Column Chart

In order to use FlexChart, we'll need to import the WjChartModule to the app.module.ts file. Like FlexGrid, we'll set the itemsSource property. We'll also set the chart header by setting the header property, the bindingX property so the chart knows what value to bind to, and the palette property, which allows us to define the colors used by the chart to display values.

There's also some more markup needed to finish filling out the chart component. The first is the wj-flex-chart-legend; the position property is set to Bottom so that the legend displays at the bottom of the chart. The second is the three series to display. This is done using the wj-flex-chart-series markup, which needs its name and binding properties set to tell the chart what to display.

Line Chart

The next most common chart type is the LineChart. Setting this up is similar to how we set up the ColumnChart. Inside of the chart markup, chartType property is set to Line so the control knows that we want to display the data as a line chart.

The other thing that we'll implement in this example is the wj-flex-chart-axis markup to set the chart's axis. Its assigned to the Y-axis by setting the wjProperty to axisY, and we the title of the axis is set by assigning a value to the title property.

Scatter Chart

The final chart we'll cover is the ScatterChart. It displays data based on both an X and Y value to place a bubble on the chart in relation to the values provided. Like the LineChart, in the markup the chart itself is defined, as well as the legend, axis, and three series. The difference is that we will not be defining the itemsSource property of the chart; instead, we define it for each of the three series.

With a ScatterChart, each of the series need to have their own itemsSource. The reason for this is because of the way the data is formatted. In the previous examples, the X-axis was bound to predefined "Year" property values. With the ScatterChart, there are no predefined values to base the chart off of. So instead, we can bind both our X and Y-axis values directly in the series markup.

Multiple Axes

FlexChart allows you to define multiple axes. This is useful when you want to display series of data that are measured by different values (for example, the hundreds vs. the hundred thousands). This is done by using a FlexChartSeries as, essentially, another FlexChart.

To set a second axis, we need to pick one of the series to base the axis on. We'll set the chartType of the series; in this case, we'll set it as LineSymbols. Then, within that series, we define the markup for another wj-flex-chart-axis. The only new thing we do with the axis is set the position property to Right.

Series Picker

There will be times where you will want to choose which series the chart is displaying; this is usually due to having many different series that are being displayed. We can allow users to select which series that they want to view by implementing a series picker.

To create the series picker, we're going to use a Wijmo ListBox. We'll also use bootstrap icons to add an icon that users can click to display the series picker. For the itemsSource, we use the series array of the FlexChart that the picker is going to be tied to. We also add methods tied to the lostFocus and checkedItemsChanged events of the control so that we can update the chart in real-time.
